# 推送展示（Broadcast Display）功能设计

**日期：** 2026/04/12
**状态：** 已批准

---

## 1. 功能概述

教师可以选择某个学生的作品，同步推送给所有学生观看。学生屏幕被全屏展示弹窗覆盖，无法执行任何操作。教师可手动关闭展示弹窗，也可单独解除锁定。

---

## 2. 交互流程

### 2.1 教师操作流程

1. 教师在学生网格中点击选中某个学生卡片（卡片高亮边框）
2. 底部控制栏出现"推送展示"按钮
3. 点击"推送展示"后：
   - 若锁定未开启，自动开启锁定
   - 向所有学生广播推送展示事件，附带该学生作品数据
4. 点击"关闭展示" → 仅关闭学生端弹窗，锁定保持
5. 点击"解除锁定" → 学生完全恢复正常操作

### 2.2 学生状态机

| 教师操作 | 学生状态 |
|---|---|
| 点击"推送展示" | 展示弹窗全屏覆盖 + 画布锁定遮罩 |
| 点击"关闭展示" | 等待提示文字 + 锁定遮罩 |
| 点击"解除锁定" | 完全恢复正常 |

### 2.3 取消选中

教师再次点击已选中的学生卡片 → 取消选中，底部"推送展示"按钮消失。

---

## 3. 视觉设计

### 3.1 展示弹窗（Broadcast Modal）

- **背景遮罩：** 半透明深色 `rgba(0, 0, 0, 0.7)`
- **内容卡片：** 白色/浅灰背景，圆角 `border-radius: 16px`，居中显示
- **学生作品：** 在卡片内居中，最大 80vw × 80vh，保持比例拉伸适配
- **底部提示：** 卡片底部一行小灰字 `"等待老师关闭"`，不显示关闭按钮
- **层级：** 覆盖在学生画布和锁定遮罩之上（z-index 高于锁定遮罩）

### 3.2 等待提示（展示关闭后）

- 全屏半透明黑色遮罩 `rgba(0, 0, 0, 0.7)`
- 遮罩中央一行白字：`"老师已关闭展示，正在等待解锁..."`

### 3.3 学生网格（教师端）

- 点击学生卡片 → 选中高亮（蓝色边框 `border: 2px solid #4A90D9`）
- 再次点击 → 取消选中

### 3.4 底部控制栏（教师端）

- 选中学生后，控制栏右侧新增"推送展示"按钮
- 按钮位于现有"锁定"/"模式切换"按钮组右侧
- 推送展示进行中时，显示"关闭展示"按钮（替换"推送展示"）

## 4. 技术实现

### 4.1 Socket.IO 事件

| 事件名 | 方向 | Payload | 说明 |
|---|---|---|---|
| `broadcast_display` | 教师→服务器→所有学生 | `{ studentId, studentName, state: { shapes, faceParts, sunSkin, canvasRadius } }` | 推送展示 |
| `display_closed` | 教师→服务器→所有学生 | 无 | 关闭展示弹窗 |
| `get_student_state` | 教师→服务器 | `{ studentId }` | 获取指定学生作品（已有，广播时复用） |

### 4.2 学生端新增状态

```js
{
  isDisplaying: boolean,      // 是否正在展示
  displayStudentName: string,  // 被展示的学生姓名
  displayState: object | null  // 被展示的作品数据
}
```

### 4.3 教师端新增状态

```js
{
  selectedStudentId: string | null,  // 选中的学生ID
  isDisplayActive: boolean            // 展示弹窗是否正在进行
}
```

| 文件 | 修改内容 |

## 5. 边界情况

- **教师在展示过程中刷新页面：** 展示状态丢失，学生仍处于锁定状态
- **被展示的学生断开连接：** 不影响展示，其他学生仍正常看到展示内容
- **教师先选中学生 A，再选中学生 B：** 新的推送会替换旧的，无需先关闭
- **锁定状态由教师手动解除，不随展示关闭自动解除**
